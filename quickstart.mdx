---
title: "Quick start"
description: "Get from zero to a running SilverBullet instance in five minutes."
---

This guide walks you through installing SilverBullet, creating your first page, and running your first query.

<Steps>
  <Step title="Install and run SilverBullet">
    Choose Docker or the single binary to get SilverBullet running locally.

    <Tabs>
      <Tab title="Docker">
        ```bash
        docker run -p 3000:3000 -v ./space:/space ghcr.io/silverbulletmd/silverbullet
        ```

        This pulls the latest image from GHCR, maps port 3000, and stores your space files in a local `./space` folder.
      </Tab>
      <Tab title="Binary">
        1. [Download the binary for your platform from GitHub releases](https://github.com/silverbulletmd/silverbullet/releases).
        2. Unzip the file and make it executable.
        3. Create a folder for your space and run the binary:

        ```bash
        mkdir space
        ./silverbullet space
        ```
      </Tab>
    </Tabs>

    <Note>
      For a full breakdown of deployment options including server setup, TLS, and authentication, see the [installation guide](/install).
    </Note>
  </Step>

  <Step title="Open SilverBullet in your browser">
    Navigate to `http://localhost:3000`. SilverBullet loads as a Progressive Web App.

    To install it as a standalone app, use the "Install" or "Add to Home Screen" option in your browser. On Chrome-based browsers, look for the install icon in the address bar. On Safari and Android, use the share menu.
  </Step>

  <Step title="Create your first page">
    Press `Ctrl-k` (or `Cmd-k` on Mac) to open the **Page Picker**. Type a name like `My First Page` and press Enter — SilverBullet creates the page immediately and opens it for editing.

    Start typing. Everything you write is Markdown.
  </Step>

  <Step title="Write Markdown and link pages">
    SilverBullet is a full Markdown editor with live preview. As you type, headings, bold text, and lists render in place.

    To link to another page, type `[[`. SilverBullet shows an autocomplete list of existing pages. Select one or type a new name and press Enter to create it. Links are **bi-directional**: the linked page will show a back-reference to the page you are writing.

    ```markdown
    # My First Page

    This links to [[Another Page]] and [[My Notes]].
    ```
  </Step>

  <Step title="Add tags and frontmatter">
    Tag a page by writing a hashtag on an empty line:

    ```markdown
    #project
    ```

    Or add structured metadata using YAML frontmatter at the top of the page:

    ```yaml
    ---
    status: active
    priority: high
    tags: project
    ---
    ```

    Tags and frontmatter attributes are automatically indexed and become queryable in the next step.
  </Step>

  <Step title="Run your first query">
    SilverBullet's Lua Integrated Query lets you generate dynamic content inline. Add this to any page:

    ```lua
    ${query[[from tags.page limit 5]]}
    ```

    Move your cursor outside the block — SilverBullet renders a live table of your pages. The result updates automatically as your space changes.

    To query only pages tagged `#project` with high priority:

    ```lua
    ${query[[from p = tags.project where p.priority == "high"]]}
    ```
  </Step>

  <Step title="Render a query with a template">
    By default, queries render as tables. You can change the output using a template. SilverBullet's standard library includes `templates.pageItem` for bullet lists:

    ```lua
    ${template.each(query[[
      from p = tags.project
      where p.priority == "high"
      order by p.lastModified desc
    ]], templates.pageItem)}
    ```

    You can also write an inline template:

    ```lua
    ${template.each(query[[
      from p = tags.project
      where p.priority == "high"
      order by p.lastModified desc
    ]], template.new[==[
        * ${name} (status: ${status})
    ]==])}
    ```
  </Step>
</Steps>
